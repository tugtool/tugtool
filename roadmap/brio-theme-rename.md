# Clean up theme file naming

*Both themes are hand-authored peers. The file names and structure should reflect that.*

---

## Why

Two vestigial names from the old base+override architecture we deleted:

1. **`tug-base-generated.css`** — This IS brio. It's hand-edited, not generated. The `generate-tug-tokens.ts` script it claims to be "generated by" doesn't even exist anymore. The "generated" name and the "do not edit" comment are lies. It should be `themes/brio.css`, living alongside `themes/harmony.css`.

2. **`tug-base.css`** — There's no "base" vs anything. It's the main CSS entry point (fonts, multipliers, motion rules, scrollbar styling, and the theme import). Should be `tug.css`.

## What

Two renames:
- `tugdeck/styles/tug-base-generated.css` → `tugdeck/styles/themes/brio.css`
- `tugdeck/styles/tug-base.css` → `tugdeck/styles/tug.css`

Remove the stale "Generated by" / "do not edit" comment from brio.css.
