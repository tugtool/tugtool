Phase 3.5C: Complete Rename Table (Four-Slot Framework)

The four-slot naming convention for formula fields
==================================================

Token names answer "what is this color used for?" with six slots:
  <plane>-<component>-<constituent>-<emphasis>-<role>-<state>

Formula fields answer "how is this color derived?" with four slots:
  <context><Constituent><State><Parameter>

Slot definitions:

  context       What design area does this knob serve?
                surface, atmosphere, content, muted, subtle, disabled,
                inverse, placeholder, onAccent, onCaution, onSuccess,
                filled, outlined, ghost, badge, field, tab, tabClose,
                toggle, icon, card, cardFrame, shadow, overlay,
                selection, highlight, border, divider, accent, caution,
                checkmark, radioDot, disabled

  constituent   What visual part? (maps to token constituent slot)
                Text, Icon, Border, Surface, Track, Thumb,
                (empty for the primary/default constituent of a context)

  state         What interaction or mode state?
                Rest, Hover, Active, Disabled, Focus, ReadOnly,
                Light (mode-specific override),
                On, Off, Mixed, Inactive,
                (empty when state-independent)

  parameter     What color dimension is this knob controlling?
                Tone, Intensity, Alpha, HueSlot, HueExpression,
                ToneOverride, ToneBase, ToneScale, ToneCenter,
                ToneComputed, Base, Scale

How six-slot token concepts map to four-slot formula concepts:

  Token plane (element/surface)   --> absent from formula names; the derivation
                                      rule determines plane, not the formula
  Token component                 --> formula context
  Token constituent               --> formula constituent
  Token emphasis                  --> formula context for controls (filled,
                                      outlined, ghost ARE the context)
  Token role                      --> absent from formula names; emphasis-level
                                      consolidation means one formula serves
                                      all roles within an emphasis
  Token state                     --> formula state
  (no token equivalent)           --> formula parameter (Tone, Intensity, etc.)


Substitution rules
==================

  bg        --> surface     (surfaces are backgrounds)
  fg        --> (context)   (the context says what kind of text: content,
                             muted, subtle, disabled, inverse, placeholder)
  atm       --> atmosphere
  txt       --> content     (the default/primary text type)
  I         --> Intensity
  Bg (mid)  --> Surface     (in compounds like outlinedBg, ghostActionBg)
  Fg (mid)  --> Text        (in compounds like outlinedFg, ghostFg, tabFg)
  Value     --> (drop)      (AlphaValue --> Alpha; ToneValue --> ToneComputed)
  Hue/Expr  --> HueExpression (for string hue dispatch fields)
  Source    --> HueExpression (fgPlaceholderSource is a hue expression)

Summary
=======

Total fields:           202 (200 after deleting 2 dead sentinel alpha fields)
Fields renamed:          ~95
Fields deleted:            2 (outlinedBgHoverAlpha, outlinedBgActiveAlpha)
Fields unchanged:       ~105

The four-slot pattern <context><Constituent><State><Parameter> explains
every field name. When reading a formula field:
  - Context tells you what part of the design system
  - Constituent tells you what visual part (Text, Icon, Border, Surface)
  - State tells you what interaction state (Rest, Hover, Active, Light)
  - Parameter tells you what color dimension (Tone, Intensity, Alpha, HueSlot)

Resolved questions:
  1. Outlined sentinel alpha collision: outlinedBgHoverAlpha and
     outlinedBgActiveAlpha are dead code — never read at runtime. The
     AlphaValue variants (outlinedBgHoverAlphaValue, outlinedBgActiveAlphaValue)
     are the real fields consumed by outlinedBg() in derivation-rules.ts.
     Resolution: DELETE the two dead fields. Rename only the AlphaValue
     variants to outlinedSurfaceHoverAlpha / outlinedSurfaceActiveAlpha.
     Field count drops from 202 to 200.
  2. bgAppSurfaceI → surfaceAppBaseIntensity. APPROVED.
  3. disabledBgBase/Scale → disabledSurfaceToneBase/ToneScale. APPROVED.
  4. bgCanvasToneSCCenter → surfaceCanvasToneCenter. APPROVED.
