## Rename #7: `separator` → `divider-separator`

**Reason:** `--tug-base-separator` has no `-border-` or `-divider-` segment. Renaming to `divider-separator` adds the `divider-` prefix, making it classifiable as an element token.

**New name:** `--tug-base-divider-separator`

**Classification after rename:** element (starts with `divider-`, consistent with `divider-default` and `divider-muted` which are already classified as element under the same rule; also explicitly listed in Spec S01 as one of the explicit element token patterns)

### Files to update

| `tugdeck/src/components/tugways/tug-inspector.css` | CSS | Line 32: `--tug-inspector-scrub-track: var(--tug-base-separator)` |
| `tugdeck/src/components/tugways/tug-dialog.css` | CSS | Line 29: `--tug-progress-track: var(--tug-base-separator)` |
| `tugdeck/src/components/tugways/tug-data.css` | CSS | Line 47: `--tug-gauge-track: var(--tug-base-separator)` |
| `tugdeck/src/__tests__/theme-derivation-engine.test.ts` | TS | Line 1195: expected value map |
| `tugdeck/scripts/generate-tug-tokens.ts` | TS | Line 122: `if (rest.startsWith("separator")) return "separator"` — must change to `if (rest.startsWith("divider-separator")) return "separator"` (or update the group name to `"divider"` to merge with the existing divider group) |

**Note on generate-tug-tokens.ts:** The `getGroup` function uses `rest.startsWith("separator")`. After rename to `divider-separator`, this match will no longer fire. The token will instead be caught by `rest.startsWith("divider-")` (line 101), which puts it in the `"divider"` group. This is actually correct behavior — `divider-separator` should be grouped with `divider-default` and `divider-muted` in the Divider group. The `"separator"` group entry in `GROUP_ORDER` and `GROUP_LABELS` will become empty and can be removed, but this is optional cleanup since an empty group emits no CSS lines.

**Decision:** Remove the `"separator"` group from `GROUP_ORDER` and `GROUP_LABELS` in `generate-tug-tokens.ts` to keep the script clean. The three divider tokens (`divider-default`, `divider-muted`, `divider-separator`) will all be emitted under the "Divider" group.
